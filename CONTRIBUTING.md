# 为 FastAPI React Starter 做贡献

首先，感谢您考虑为 FastAPI React Starter 做贡献！正是像您这样的人使这个项目成为一个如此优秀的工具。

## 行为准则

通过参与本项目，您需要遵守我们的[行为准则](./CODE_OF_CONDUCT.md)。

## 我如何贡献？

### 报告错误

本节指导您如何提交错误报告。遵循这些指南有助于维护者和社区理解您的报告、重现行为并找到相关报告。

- 使用 GitHub 问题搜索 — 检查问题是否已被报告。
- 检查问题是否已修复 — 尝试使用最新的 `main` 分支重现它。
- 创建问题时使用错误报告模板。

### 建议增强功能

本节指导您如何提交增强建议，包括全新的功能和现有功能的微小改进。

- 使用 GitHub 问题搜索 — 检查增强功能是否已被建议。
- 创建问题时使用功能请求模板。

### 拉取请求

- 填写必需的模板
- 不要在 PR 标题中包含问题编号
- 遵循风格指南
- 包含适当的测试用例
- 为新代码编写文档

## 风格指南

### Git 提交消息

我们使用遵循以下指南的 AI 生成的提交消息：

- 以命令式形式的短句开头，不超过 50 个字符
- 留一个空行，然后继续更详细的说明
- 第一部分应该只是一句话
- 详细说明最多应该是两到三句话
- 避免过于冗长的描述或不必要的细节

示例：

```
添加用户身份验证功能

实现基于 JWT 的身份验证系统，包含登录和注册端点。
添加用于受保护路由和用户会话管理的中间件。
```

### JavaScript 风格指南

- 使用分号
- 使用 2 个空格缩进
- 优先使用 `const` 而不是 `let` 或 `var`
- 使用有意义的变量名

### Python 风格指南

- 遵循 PEP 8
- 使用 4 个空格缩进
- 使用有意义的变量名

## 附加说明

### 问题和拉取请求标签

本节列出了我们用于帮助跟踪和管理问题和拉取请求的标签。

- `bug` - 表示代码库中错误的问题
- `documentation` - 与文档相关的问题
- `enhancement` - 表示新功能请求的问题
- `good first issue` - 适合新手的良好问题
- `help wanted` - 需要社区帮助的问题
