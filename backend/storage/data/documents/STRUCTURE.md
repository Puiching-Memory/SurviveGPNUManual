# 文档结构说明

本文档说明文档的组织结构和分类原则。

## 目录结构

```
documents/
├── guides/          # 核心指南文档（手册主体）
│   ├── home.md      # 首页/序章
│   ├── daily.md     # 校园生活
│   ├── exam.md      # 学业指南
│   ├── tips.md      # 实用技巧
│   ├── data.md      # 数据资料
│   ├── community.md # 社区指南
│   └── README.md    # 文档说明
├── career/          # 职业与升学规划
│   ├── job.md       # 求职指南
│   ├── aspiration.md # 升学规划
│   ├── postgraduate.md # 读研指南
│   └── after.md     # 毕业后指南
└── blog/            # 博客文章
    ├── echo-*.md    # 个人感悟
    ├── git-usage.md # 技术教程
    ├── photoTop-*.md # 图片集
    └── tieba-*.md   # 贴吧相关文章
```

## 分类原则

### guides/ - 核心指南文档

手册的主体内容，面向所有用户的核心指南：

- **home.md**: 首页/序章，手册介绍
- **daily.md**: 校园生活指南（校历、宿舍、食堂等）
- **exam.md**: 学业相关指南（课程、考试等）
- **tips.md**: 实用技巧（邮箱、VPN、系统使用等）
- **data.md**: 数据资料库
- **community.md**: 社区参与指南（如何贡献、提交问题等）

**特点**: 
- 结构化、系统化的指南内容
- 需要持续维护和更新
- 面向所有用户

### career/ - 职业与升学规划

关于职业发展和升学的专门指南：

- **job.md**: 求职指南（简历、校招、实习等）
- **aspiration.md**: 升学规划（学校排名、升学路径等）
- **postgraduate.md**: 读研指南（保研、考研等）
- **after.md**: 毕业后指南

**特点**:
- 面向特定用户群体（即将毕业或规划未来）
- 相对独立的内容模块
- 可按需扩展子分类

### blog/ - 博客文章

社区生成的内容，个人分享和经验交流：

- **echo-*.md**: 个人感悟、实习经历等
- **git-usage.md**: 技术教程
- **photoTop-*.md**: 图片集
- **tieba-*.md**: 来自贴吧的文章

**特点**:
- 内容多样，风格自由
- 按时间或主题组织
- 可包含个人观点和经历分享

## 添加新文档的指南

### 判断文档应该放在哪里？

1. **核心指南内容** → `guides/`
   - 手册的核心章节
   - 所有用户都需要了解的内容
   - 需要系统化组织的知识

2. **职业/升学相关内容** → `career/`
   - 求职、升学、职业规划
   - 面向特定阶段的学生
   - 相对独立的主题模块

3. **个人分享/经验交流** → `blog/`
   - 个人经历、感悟
   - 技术教程、经验分享
   - 来自社区的内容

### 命名规范

- 使用小写字母和连字符：`example-document.md`
- 文件名应该简洁、描述性
- Blog 文章可以使用日期前缀：`2025-01-20-title.md`

## 扩展建议

未来可以考虑的子分类：

- `guides/campus/` - 校区相关详细指南
- `guides/academic/` - 学术相关详细指南
- `career/internship/` - 实习相关
- `career/study-abroad/` - 留学相关
- `blog/tutorials/` - 技术教程
- `blog/stories/` - 个人故事
